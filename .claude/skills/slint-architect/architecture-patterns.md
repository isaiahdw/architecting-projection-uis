# Slint Component Architecture Patterns for Projection

## Core Architecture

Projection applications have three runtime layers:

```
+------------------+     stdio {:packet, 4}     +------------------+
|    Elixir BEAM   | <========================> |   Rust ui_host   |
|  (state + logic) |   length-prefixed JSON     | (bridge + render)|
+------------------+                            +--------+---------+
                                                         |
                                                    Slint runtime
                                                   (GPU rendering)
```

- **Elixir** owns all application state and business logic
- **Rust** bridges the two — decodes envelopes, applies patches to Slint properties
- **Slint** renders the UI and captures user input as intents

## Component Hierarchy

### Top-Level Window

The `AppWindow` is generated by the codegen task. It contains:

```slint
// Generated — do not edit directly
export component AppWindow inherits Window {
    in property <string> app_title: "Projection";
    in property <string> active_screen: "hello";
    in property <bool> nav_can_back: false;

    callback ui_intent(string, string);
    callback navigate(string, string);

    // AppShell with navigation chrome
    // Screen switcher based on active_screen
    // ErrorScreen fallback
}
```

### The Screen Base Component

Every screen inherits from `Screen`, which inherits `VerticalLayout`:

```slint
import { UI } from "ui.slint";

export component Screen inherits VerticalLayout {
    callback intent(intent_name: string, intent_arg: string);
    intent(intent_name, intent_arg) => {
        UI.intent(intent_name, intent_arg);
    }
}
```

This means:
- All screens are vertical layouts by default
- `root.intent(name, arg)` is available in every screen to send events to Elixir
- The intent bubbles up through the `UI` global to the Rust host

### The UI Global

```slint
export global UI {
    callback intent(intent_name: string, intent_arg: string);
}
```

Single exit point for all user intents. The Rust `HostBindings` trait binds this
callback to the outbound envelope queue.

### AppShell (Navigation Chrome)

```slint
export component AppShell inherits Rectangle {
    in property <string> app_title: "Projection";
    in property <bool> show_back: false;
    in property <length> window_width: 480px;
    in property <length> window_height: 320px;
    callback nav_back();

    background: #1a1a2e;

    VerticalLayout {
        // Header with optional back button + title
        // Separator line
        // @children — active screen content goes here
    }
}
```

### ErrorScreen (Fallback)

```slint
import { Screen } from "screen.slint";

export component ErrorScreen inherits Screen {
    in property <string> title: "Rendering Error";
    in property <string> message: "";
    in property <string> screen_module: "";
    // Displays error details when a screen fails to render
}
```

## Layout Patterns for Embedded Displays

### Pattern 1: Fixed-Size Dashboard

For small embedded displays (e.g., 480x320):

```slint
export component DashboardScreen inherits Screen {
    in property <string> title: "Status";
    in property <float> temperature: 0.0;
    in property <bool> connected: false;

    padding: 8px;
    spacing: 6px;

    // Title row
    Text {
        text: root.title;
        font-size: 16px;
        font-weight: 700;
        color: #f2f2ff;
    }

    // Metric cards in a grid
    GridLayout {
        spacing: 8px;
        Row {
            MetricCard { label: "Temp"; value: root.temperature + " C"; }
            MetricCard { label: "Status"; value: root.connected ? "Online" : "Offline"; }
        }
    }
}
```

### Pattern 2: Scrollable List

For screens with variable-length content:

```slint
export component DeviceListScreen inherits Screen {
    in property <[string]> device_names: [];

    padding: 8px;
    spacing: 4px;

    Text {
        text: "Devices (" + device_names.length + ")";
        font-size: 16px;
        color: #f2f2ff;
    }

    Flickable {
        viewport-height: device_names.length * 40px;

        VerticalLayout {
            spacing: 4px;
            for name[index] in root.device_names: Rectangle {
                height: 36px;
                border-radius: 4px;
                background: #232342;

                HorizontalLayout {
                    padding-left: 12px;
                    Text {
                        text: name;
                        color: #d8d8f0;
                        vertical-alignment: center;
                    }
                }

                TouchArea {
                    clicked => {
                        root.intent("device.select", index);
                    }
                }
            }
        }
    }
}
```

### Pattern 3: Tab-Style Navigation

When you need sub-navigation within a screen:

```slint
export component SettingsScreen inherits Screen {
    in property <string> active_tab: "general";
    in property <string> device_name: "";
    in property <string> wifi_ssid: "";

    padding: 8px;
    spacing: 8px;

    // Tab bar
    HorizontalLayout {
        height: 32px;
        spacing: 4px;

        TabButton { label: "General"; active: root.active_tab == "general";
                    clicked => { root.intent("settings.tab", "general"); } }
        TabButton { label: "Network"; active: root.active_tab == "network";
                    clicked => { root.intent("settings.tab", "network"); } }
    }

    // Content area — conditional on active tab
    if root.active_tab == "general": VerticalLayout {
        Text { text: "Device: " + root.device_name; color: #d8d8f0; }
    }

    if root.active_tab == "network": VerticalLayout {
        Text { text: "WiFi: " + root.wifi_ssid; color: #d8d8f0; }
    }
}
```

### Pattern 4: Form with Input

Slint's `TextInput` for embedded forms:

```slint
export component EditScreen inherits Screen {
    in property <string> field_value: "";

    padding: 12px;
    spacing: 8px;

    Text { text: "Edit Name"; font-size: 16px; color: #f2f2ff; }

    Rectangle {
        height: 36px;
        border-radius: 4px;
        background: #232342;
        border-width: 1px;
        border-color: input.has-focus ? #5566cc : #333355;

        input := TextInput {
            text: root.field_value;
            color: #d8d8f0;
            font-size: 14px;
            horizontal-alignment: left;
            vertical-alignment: center;

            accepted => {
                root.intent("edit.save", self.text);
            }
        }
    }

    Rectangle {
        height: 34px;
        border-radius: 6px;
        background: #335533;

        Text {
            text: "Save";
            horizontal-alignment: center;
            vertical-alignment: center;
            color: #ccffcc;
        }

        TouchArea {
            clicked => {
                root.intent("edit.save", input.text);
            }
        }
    }
}
```

## Slint Property-to-Schema Type Mapping

| Slint Declaration | Projection Schema | Notes |
|-------------------|-------------------|-------|
| `in property <string> name: "";` | `field :name, :string` | Direct mapping |
| `in property <bool> active: false;` | `field :active, :bool` | Direct mapping |
| `in property <int> count: 0;` | `field :count, :integer` | Slint uses `int` |
| `in property <float> temp: 0.0;` | `field :temp, :float` | Direct mapping |
| `in property <[string]> items: [];` | `field :items, :list, items: :string` | Array syntax |
| `in property <[int]> ids: [];` | `field :ids, :list, items: :integer` | Typed items |
| N/A (codegen setters) | `field :data, :id_table, columns: [...]` | No direct Slint equivalent |
| N/A (flattened prefix) | `component :badge, StatusBadge` | Generates `badge_label`, `badge_active` |

## Slint Color System

For embedded displays, use a consistent dark theme:

```
Background layers:
  #1a1a2e  — deepest background (app shell)
  #232342  — card/surface background
  #2a2a4a  — hover state / separator lines
  #333355  — borders, subtle outlines

Text hierarchy:
  #f2f2ff  — primary text (titles, headings)
  #d8d8f0  — body text
  #cfcfe7  — button text
  #c0c0d0  — secondary text
  #b8b8d0  — muted text
  #666688  — tertiary/debug text

Status colors:
  #44cc44 / #88cc88  — success / online
  #cc4444            — error / danger
  #ccaa44            — warning
  #5566cc            — accent / focus
```

## Animation Patterns

For embedded targets, keep animations simple and performant:

```slint
// Fade transition
export component FadePanel inherits Rectangle {
    in property <bool> visible: true;
    opacity: visible ? 1.0 : 0.0;
    animate opacity { duration: 200ms; easing: ease-in-out; }
}

// Color transition for status indicators
export component StatusDot inherits Rectangle {
    in property <bool> active: false;
    width: 12px;
    height: 12px;
    border-radius: 6px;
    background: active ? #44cc44 : #cc4444;
    animate background { duration: 300ms; }
}
```

**Embedded animation rules:**
- Keep durations under 300ms
- Prefer `opacity` and `background` animations (GPU-friendly)
- Avoid animating `width`/`height` on complex layouts (triggers relayout)
- Use `ease-in-out` as default easing for smooth feel
- Test on target hardware — frame budget matters

## Conditional Rendering

Slint supports `if` for conditional elements:

```slint
// Show/hide based on state
if root.is_loading: Text { text: "Loading..."; color: #b8b8d0; }

if !root.is_loading: VerticalLayout {
    // Main content
}

// Ternary for property values
Text {
    text: root.count > 0 ? "Items: " + root.count : "No items";
    color: root.count > 0 ? #88cc88 : #cc4444;
}
```

## Repeater Pattern (for loops)

```slint
for item[index] in root.items: Rectangle {
    height: 32px;
    Text { text: item; }
    TouchArea {
        clicked => { root.intent("item.select", index); }
    }
}
```

Note: `index` is zero-based. The intent arg is always a `string` — convert
numeric indices with string interpolation in Slint or parse in Elixir.

## Slint Language Quick Reference

### Naming Conventions

| Context | Convention | Example |
|---------|-----------|---------|
| Components | PascalCase | `DashboardScreen`, `StatusBadge` |
| Properties | kebab-case | `font-size`, `border-radius`, `horizontal-alignment` |
| Callbacks | kebab-case | `value-changed`, `item-clicked` |
| Element IDs | kebab-case | `my-button := TouchArea { }` |
| Globals | PascalCase | `global UI { }`, `global AppState { }` |
| Structs/Enums | PascalCase | `struct SensorData { }`, `enum Status { }` |

**Note:** Projection's `.slint` files use snake_case for properties that map to
Elixir schema fields (e.g., `click_count`, `cpu_temp`). This is intentional —
the codegen bridge handles the mapping.

### Property Qualifiers

```slint
in property <string> label: "";       // Set by parent, read-only inside
out property <bool> pressed: false;    // Set inside, read-only from outside
in-out property <int> counter: 0;      // Readable and writable from both sides
property <float> internal: 0.0;        // Private — not accessible from outside
```

In Projection screens, always use `in property` for fields bound from Elixir.

### String Interpolation

```slint
// Slint uses \{} for string interpolation (NOT ${} like JavaScript)
Text { text: "Count: \{root.count}"; }
Text { text: "Temp: \{Math.round(root.temperature)} C"; }

// Simple concatenation with + also works (used in Projection templates)
Text { text: "Status: " + root.status; }
Text { text: root.cpu_temp + " C"; }
```

### Element References

```slint
// Use := to name an element for referencing
input := TextInput { text: "hello"; }

// Special references:
// root  — the component's root element
// self  — the current element
// parent — the parent element
```

### Key Built-In Types

| Type | Slint Syntax | Example |
|------|-------------|---------|
| Integer | `int` | `42` |
| Float | `float` | `3.14` |
| String | `string` | `"hello"` |
| Boolean | `bool` | `true` |
| Color | `color` | `#ff0000`, `#AARRGGBB` |
| Length | `length` | `100px`, `2cm` |
| Duration | `duration` | `250ms`, `1s` |
| Image | `image` | `@image-url("icon.png")` |
| Array | `[T]` | `[string]`, `[int]` |

**Color gotcha:** Hex colors with alpha use `#AARRGGBB` format — alpha comes
FIRST, not last.

### Standard Widgets (import from "std-widgets.slint")

```slint
import { Button, LineEdit, TextEdit, ComboBox, CheckBox, Switch,
         Slider, SpinBox, ProgressIndicator, ListView, ScrollView,
         StandardListView, StandardTableView, TabWidget, GroupBox,
         VerticalBox, HorizontalBox, GridBox, Spinner } from "std-widgets.slint";
```

**Note:** Projection templates do NOT use std-widgets by default — they use
custom components for full control over embedded rendering. You can import
std-widgets if your target supports one of the built-in themes (fluent,
material, cupertino).

### States and Transitions

```slint
component ToggleButton inherits Rectangle {
    in property <bool> active: false;

    background: #232342;

    states [
        active-state when root.active: {
            root.background: #335533;
        }
    ]

    transitions [
        in active-state: {
            animate root.background { duration: 200ms; easing: ease-out; }
        }
        out active-state: {
            animate root.background { duration: 150ms; easing: ease-in; }
        }
    ]
}
```

### Key Gotchas

1. **`visible: false` still occupies layout space** — use `if condition:` for
   true conditional inclusion/exclusion from layout

2. **No imperative control flow in bindings** — property bindings are reactive
   expressions; use callbacks for imperative logic

3. **Image paths are relative to the .slint file**, not the working directory

4. **Inheritance gives built-in properties only** — when you write
   `component Foo inherits Rectangle`, you get Rectangle's properties, but
   custom properties from intermediate components must be re-declared

5. **The `init` callback** runs once on instantiation:
   ```slint
   component MyComp { init => { debug("created"); } }
   ```

6. **Two-way binding syntax**: `property <=> other.property`

7. **No `else` keyword** — use separate `if` blocks with opposite conditions:
   ```slint
   if root.loading: Text { text: "Loading..."; }
   if !root.loading: Text { text: "Ready"; }
   ```

8. **`pure` keyword** — Mark functions/callbacks as side-effect-free to use in
   reactive bindings:
   ```slint
   pure function format-temp(value: float) -> string { ... }
   ```

9. **Function visibility** — `public` (any parent), `protected` (inheritors only),
   default is private (component-internal only)

10. **Callbacks vs functions** — Use callbacks when the handler is implemented in
    backend code (Rust). Use functions for Slint-internal logic.

## Focus Handling

Critical for embedded devices with keyboard, encoder, or D-pad input:

```slint
export component FormScreen inherits Screen {
    // forward-focus on the component sets initial focus when screen loads
    forward-focus: name-input;

    in property <string> name_value: "";

    padding: 12px;
    spacing: 8px;

    Text { text: "Name:"; color: #b8b8d0; }

    name-input := TextInput {
        text: root.name_value;
        color: #d8d8f0;
        font-size: 14px;

        accepted => {
            root.intent("form.save", self.text);
        }
    }

    // Programmatic focus control
    Rectangle {
        height: 34px;
        border-radius: 6px;
        background: #232342;

        Text { text: "Edit Name"; horizontal-alignment: center; vertical-alignment: center; color: #cfcfe7; }

        TouchArea {
            clicked => {
                name-input.focus();  // Activate focus programmatically
            }
        }
    }
}
```

**Key APIs:**
- `element.focus()` — Programmatically activate focus on an element
- `element.clear-focus()` — Remove focus; keyboard input discarded until refocus
- `forward-focus: element-id` — Delegate focus to a child (use on component or Window)
- `has-focus` — Read-only property indicating current focus state

**Embedded input patterns:**
- Use `FocusScope` for keyboard/encoder navigation between custom controls
- Use `forward-focus` on screen components to set initial focus on load
- Style focused elements with `has-focus` conditional colors/borders

## Custom Font Embedding

On embedded targets, system fonts may not exist. Import fonts directly:

```slint
// Import font file — registers all families globally
import "./fonts/NotoSans-Regular.ttf";
import "./fonts/NotoSans-Bold.ttf";

export component AppWindow inherits Window {
    // Set default font for all text in the window
    default-font-family: "Noto Sans";
    default-font-size: 14px;

    Text {
        text: "Uses Noto Sans Regular";
    }

    Text {
        text: "Bold text";
        font-weight: 700;  // Uses NotoSans-Bold.ttf
    }
}
```

**Supported formats:** `.ttf`, `.ttc`, `.otf`

**Embedded rules:**
- Always import fonts explicitly — don't rely on system fonts
- Use `default-font-family` on Window to set the global default
- For MCU targets, subset fonts to needed glyphs at build time (via
  `EmbedResourcesKind::EmbedForSoftwareRenderer` in Rust `build.rs`)
- Import only the weights you use — each font file adds to binary size

## Debugging Tools

Slint provides environment variables for debugging during development:

| Variable | Value | Purpose |
|----------|-------|---------|
| `SLINT_SLOW_ANIMATIONS` | Integer factor (e.g., `4`) | Slow down all animations globally |
| `SLINT_SCALE_FACTOR` | Float (e.g., `2.0`) | Simulate different pixel ratios |
| `SLINT_DEBUG_PERFORMANCE` | `refresh_lazy,overlay` | Show frame rate on screen |
| `SLINT_BACKEND` | `winit-software` | Force a specific renderer |

**`debug()` function** — Print property values to stderr:
```slint
TouchArea {
    clicked => { debug("clicked at", self.mouse-x, self.mouse-y); }
}
```

**Performance monitoring modes:**
- `refresh_lazy` — Measure FPS only during actual visual changes
- `refresh_full_speed` — Force continuous refresh to find bottlenecks
- `console` — Report to stderr
- `overlay` — Display FPS as on-screen text

Combine: `SLINT_DEBUG_PERFORMANCE=refresh_full_speed,overlay`

**`cache-rendering-hint`** — Render complex elements to texture (trades GPU
memory for draw call reduction):
```slint
Rectangle {
    cache-rendering-hint: true;
    // Complex child elements cached as texture
}
```

## Drop Shadows

Slint supports drop shadows on `Rectangle` elements:

```slint
Rectangle {
    width: 200px;
    height: 100px;
    background: #232342;
    border-radius: 8px;

    drop-shadow-blur: 8px;
    drop-shadow-color: #00000040;
    drop-shadow-offset-x: 2px;
    drop-shadow-offset-y: 4px;
}
```

| Property | Type | Default | Purpose |
|----------|------|---------|---------|
| `drop-shadow-blur` | length | 0px | Blur radius (0 = sharp edge) |
| `drop-shadow-color` | color | transparent | Shadow color (use alpha for subtlety) |
| `drop-shadow-offset-x` | length | 0px | Horizontal displacement |
| `drop-shadow-offset-y` | length | 0px | Vertical displacement |

**Renderer support:** Drop shadows work on **FemtoVG** and **Skia** renderers
(embedded Linux with GPU, macOS for testing). They are **not supported on the
software renderer** (MCU targets) — the properties are silently ignored.

## Software Renderer Limitations (MCU only)

These apply **only** to the software renderer (MCU/no-GPU targets). If your
embedded target has a GPU (embedded Linux with OpenGL ES 2.0+), use FemtoVG
or Skia and all features — including drop shadows, rotation, and clipping — work fully.

Software renderer restrictions: no rotation/scaling, no `drop-shadow-*`, no
`border-radius` + `clip: true`, no text stroking, no `Layer` element.
